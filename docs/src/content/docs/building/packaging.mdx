title: Packaging
description: How to package Flaggi for distribution on macOS, Windows, and other platforms.

The `scripts/package.sh` script creates native installers with an embedded minimal JRE — end users don't need Java installed.

## Usage

```bash
scripts/package.sh [client] [--version <ver>] [--skip-build] [--help]
```

| Flag | Description |
|------|-------------|
| `--version <ver>` | Override version number (default from build) |
| `--skip-build` | Skip the Gradle build step |
| `--help` | Show usage |

Output is written to the `dist/` directory.

## Process

The packaging pipeline:

1. **Build shadowJar** — create a fat JAR with all dependencies
2. **jdeps** — analyze the JAR to determine required JVM modules
3. **jlink** — build a custom minimal JRE with only the needed modules
4. **jpackage** — create a native installer wrapping the JAR + custom JRE

This produces a self-contained installer that doesn't require Java on the target machine.

## macOS

**Requirements**: `create-dmg` (`brew install create-dmg`)

**Output**: `.dmg` installer with a draggable `.app` icon

The script:
1. Runs `jpackage --type app-image` to create a `.app` bundle
2. Uses `create-dmg` to wrap it in a `.dmg` with a nice installer window

## Windows

**Requirements**: [WiX 3](https://github.com/wixtoolset/wix3/releases/latest)

**Important**: jpackage does **not** support WiX 4+. You must use WiX 3.

**Environment variables** needed:
- `JAVA_HOME` — path to JDK 21
- `WIX` — path to WiX 3 installation

**Output**: `.exe` installer with:
- Start menu shortcut
- Desktop shortcut
- Registry entries for uninstall

The script auto-detects the OS using `uname -s` and supports Cygwin/WSL path conversions.

## GitHub Releases

Release artifacts are uploaded to [GitHub Releases](https://github.com/matejstastny/flaggi/releases):

| Asset | Platform | Description |
|-------|----------|-------------|
| `Flaggi.dmg` | macOS | Native installer |
| `Flaggi.exe` | Windows | Native installer |
| `Flaggi-universal.jar` | Any (needs Java) | Cross-platform fat JAR |
| `Flaggi-server.jar` | Any (needs Java) | Server fat JAR |

## Adding new platforms

To add Linux or other platforms:

1. Add a new case in `package.sh` for the platform (detected via `uname -s`)
2. Use `jpackage` with the appropriate `--type` flag:
   - Linux: `--type deb` or `--type rpm`
   - Generic: `--type app-image` (no installer, just a directory)
3. Update CI workflows to build on the new platform
