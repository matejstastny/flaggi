`GameManager` (`flaggi.server.common`) runs one game instance. It processes player input, simulates physics, handles collisions, manages flag mechanics, and sends state updates.

## Game tick

Each tick of `GameManager.update()`:

1. **Process input** ‚Äî read latest `ClientStateUpdate` per player
2. **Move players** ‚Äî apply movement with collision detection
3. **Update animations** ‚Äî set sprite state based on movement direction
4. **Spawn bullets** ‚Äî create bullet if player is shooting and cooldown expired
5. **Tick bullets** ‚Äî advance positions, check tree and player collisions
6. **Tick flags** ‚Äî check pickup, carry, capture, and return logic
7. **Send state** ‚Äî build per-player `ServerStateUpdate` and send via UDP

## Movement

Player speed is `3.0 pixels per tick`. Movement uses a sliding collision system:

1. **Try diagonal** ‚Äî move in both X and Y
2. **If blocked, try horizontal only** ‚Äî move in X
3. **If blocked, try vertical only** ‚Äî move in Y
4. **Clamp** ‚Äî keep player within room boundaries (0 to roomWidth/Height)

```
tryMove(player, dx, dy, obstacles):
  newX = player.x + dx
  newY = player.y + dy
  for each obstacle:
    if AABB overlaps(player at newPos, obstacle):
      return false
  player.setX(newX)
  player.setY(newY)
  return true
```

## Collision detection

All collisions use **AABB** (Axis-Aligned Bounding Box) checks. Each game object has a hitbox defined by offset + dimensions relative to its origin:

```
worldLeft   = x + collisionOffsetX
worldTop    = y + collisionOffsetY
worldRight  = worldLeft + collisionWidth
worldBottom = worldTop + collisionHeight
```

Two objects overlap when their world-space bounding boxes intersect on both axes.

## Combat

### Shooting

- Players press `KEY_SHOOT` to fire
- Cooldown: `400ms` between shots (`SHOOT_COOLDOWN_MS`)
- Bullets spawn at the player's position, aimed toward the mouse cursor
- Direction is normalized and scaled by `BULLET_SPEED = 8.0 px/tick`

### Bullets

Each tick:
1. Advance bullet position by its velocity vector
2. Check collision with trees ‚Üí destroy bullet
3. Check collision with players (not the bullet's owner) ‚Üí apply damage, destroy bullet
4. Check lifetime (max 120 ticks = 2 seconds) and room bounds ‚Üí destroy if expired

### Damage

- Bullet damage: `25.0`
- Player max HP: `100.0`
- **4 hits to kill**
- On death: instant respawn at spawn point, full HP restore, drop any carried flag

## Flag mechanics

Each game has two flags: RED and BLUE. The flag state machine:

```

```

### Rules

- **Pickup**: A player touching the *enemy* flag picks it up (AT_BASE or DROPPED ‚Üí CARRIED)
- **Carry**: The flag follows the carrier's position
- **Capture**: The carrier touches their *own* flag's base ‚Üí score + both flags return to base
- **Drop**: If the carrier dies, the flag drops at their position (CARRIED ‚Üí DROPPED)
- **Return**: A player touching their *own* dropped flag returns it to base (DROPPED ‚Üí AT_BASE)

### Win condition

First team to **3 captures** wins. The server sends `ServerEndGame` to both clients with the winner's name.

## State updates

After all logic runs, `GameData.getServerUpdateData()` builds personalized `ServerStateUpdate` messages for each player:
- `me` ‚Äî the player's own `ServerGameObject`
- `other` ‚Äî all other objects (opponent, trees, bullets, flags)
- `tick` ‚Äî current tick number

Carried flags are excluded from the `other` list since their position is implicit (same as carrier).
